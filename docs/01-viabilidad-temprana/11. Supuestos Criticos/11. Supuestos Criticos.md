### 3. Vínculo con MVP
Los experimentos Críticos dictan que el MVP (D12) **debe** ser un producto de software funcional (Programado en Java y JS). No podemos simular flujos de Webhooks asíncronos y latencias de base de datos a escala usando solo humanos. Se requiere un MVP programado con una base de datos relacional para probar el modelo de concurrencia y oráculo manual.
