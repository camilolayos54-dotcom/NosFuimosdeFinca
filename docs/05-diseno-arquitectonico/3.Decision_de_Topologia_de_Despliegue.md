### 2. Registro de Decisión Arquitectónica (ADR)

**ADR-005: Cloud Deployment Topology**

- **Contexto:** Necesitamos definir dónde hospedar la plataforma Políglota (Next.js + Spring Boot) para cumplir con los NFRs de escalabilidad y alta disponibilidad estipulados en D1, mientras mantenemos un "Time-to-Market" rápido.
- **Opciones Consideradas:** 
  1. Kubernetes Nativo (EKS) para todo el ecosistema.
  2. Plataforma Híbrida: Vercel (Front) + AWS Managed Services (Back).
  3. Todo en Vercel (Next.js Serverless Backend).
- **Decisión:** Se elige la Opción 2 (**Plataforma Híbrida: Vercel + AWS**). El frontend residirá en Vercel Edge Network. El Backend (Monolito Spring Boot) correrá en AWS Elastic Beanstalk (o ECS), anclado a un Amazon RDS PostgreSQL gestionado.
- **Consecuencias:** 
  - *Positivo:* Next.js rinde a velocidad máxima en Vercel (SSR y Edge Caching). RDS PostgreSQL garantiza la integridad ACID para evitar Double Bookings sin lidiar con parches de SO. 
  - *Negativo:* Introduce latencia de red (aprox 20-30ms) entre Vercel y el VPC de AWS, requiriendo encriptación TLS en tránsito obligatoria.
