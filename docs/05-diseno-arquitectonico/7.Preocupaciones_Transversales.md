# Entregable 7 (D7): Preocupaciones Transversales

**Proyecto:** Nos Fuimos de Finca
**Fase:** 5 — Diseño Arquitectónico
**Estado:** Aprobado

## 1. Estrategia de Seguridad
- **Autenticación:** Módulo M-01 delegará el cifrado de contraseñas a **Bcrypt**. Los tokens de sesión serán JWT firmados asimétricamente (RS256) emitidos por el backend.
- **Autorización:** Verificación de roles (RBAC) inyectada a nivel de Filtro/Interceptor en Spring Boot y middlewares en Next.js (Edge middleware).
- **Privacidad de Datos:** Todas las peticiones HTTP estarán forzadas a HTTPS (TLS 1.3) vía CloudFront y Vercel Edge Network. Las variables sensibles (ej. `WOMPI_EVENT_SECRET`) jamás se inyectarán en el código, solo en las Variables de Entorno del servidor (`.env` / AWS Parameter Store).

## 2. Observabilidad y Logging
- **Logs de Aplicación:** El monolito de Spring Boot utilizará SLF4J + Logback para imprimir JSON logs por `stdout`. En la infraestructura AWS ECS/Elastic Beanstalk, estos logs viajarán automáticamente a Amazon CloudWatch.
- **Trazabilidad:** Cada petición HTTP entrante se le inyectará un `X-Request-ID` para trazarla a través de los logs si falla.

## 3. Manejo de Errores
- **API Backend:** Centralización de excepciones vía `@ControllerAdvice` (Spring). Todo error arrojará un payload estándar estandarizado (RFC 7807 Problem Details for HTTP APIs).
  ```json
  {
    "type": "https://api.nosfuimosdefinca.com/errors/conflict",
    "title": "Double Booking Detected",
    "status": 409,
    "detail": "Fechas ya ocupadas"
  }
  ```
- **UI Frontend:** Uso de "Error Boundaries" en React para evitar que la app entera crashee si un solo componente falla.
