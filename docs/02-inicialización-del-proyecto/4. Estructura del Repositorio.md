### 2. Tipo de Repositorio

|Campo|Valor|
|---|---|
|Repository Type|`monorepo políglota (particionado)`|
|Justification|Dada la tecnología con gestores de paquetes divergentes (NPM vs Maven), separarlo rompería la cohesión histórica. Un monorepo en raíz los orquesta unificados.|
|Restricciones Heredadas|Ninguno|

### 3. Directorios de Nivel Superior

|Directorio|Propósito|Decisión D3 que Refleja|
|---|---|---|
|`docs/`|Documentación del proyecto y ADRs|N/A|
|`frontend/`|Interfaz de Usuario (B2C Checkout & B2B Panel)|Next.js + TailwindCSS|
|`backend/`|API Transaccional y Webhooks|Java (Spring Boot)|

### 4. Convenciones de Nomenclatura

|Elemento|Convención|Tipo|
|---|---|---|
|carpetas y archivos raíz|`lowercase-hyphen`|decisión de equipo|
|paquetes backend|`lowercase`|impuesto por el framework (Java)|
|componentes UI|`PascalCase`|impuesto por el framework (React)|

### 5. Reglas de Configuración

|Archivo|Ubicación|Ignorado por Git|Notas|
|---|---|---|---|
|`.env.example`|`frontend/.env.example`, `backend/.env.example`|No|Contiene llaves, sin valores|
|`.env`|`frontend/.env`, `backend/.env`|Sí|Todas las instancias|

### 6. Estrategia de Migración

|Campo|Valor|
|---|---|
|Migration Directory|`backend/src/main/resources/db/migration/`|
|Naming Convention|`V1__init.sql` (Flyway/Liquibase standard)|
|Immutability Rule|después del despliegue: nunca modificar|
|Not Applicable|Ninguno|

### 8. Árbol de Directorios Anotado

```text
/nos-fuimos-finca-repo
├── docs/                # Documentación del proyecto y ADRs
├── frontend/            # Interfaz de Usuario B2C & B2B (Next.js)
│   ├── app/             # App Router
│   ├── components/      # UI components (TailwindCSS)
│   ├── lib/             # Fetch API clients
│   └── package.json     # Node deps
└── backend/             # API Transaccional y Webhooks (Spring Boot)
    ├── src/main/java/   # Lógica (Controllers, Services)
    ├── src/main/resources/ # Properties, DB Migrations
    └── pom.xml          # Maven deps
```
