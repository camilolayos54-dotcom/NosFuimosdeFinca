### 2. Definiciones de Entorno

|Entorno|Propósito|Roles de Acceso|Tipo de Acceso|
|---|---|---|---|
|Desarrollo|Prototipado rápido con BD local efímera y mock data.|`developer`|`deploy / admin`|
|Staging|Pruebas de concurrencia y validación de Webhooks contra Sandbox de Wompi.|`developer, qa`|`read / deploy`|
|Producción|Entorno transaccional en vivo con llaves reales de pasarela (dinero real).|`lead_engineer, devops`|`admin`|

### 3. Reglas de Promoción

|De|Para|Condición|
|---|---|---|
|Desarrollo|Staging|Pull Request creado. Vercel Preview generado con éxito. Linters y Unit Tests en verde en CI.|
|Staging|Producción|PR Aprobado por Lead. CI pasa pruebas de JMeter de concurrencia. Merge a `main` completo.|

### 4. Fuentes de Configuración

|Entorno|Fuente de Configuración|Diferencias con Producción|
|---|---|---|
|Desarrollo|Local `.env` file|BD Docker local, llaves Sandbox ficticias.|
|Staging|Vercel / Render Environment Variables|Apunta a BD Cloud ofuscada y Sandbox de Wompi.|
|Producción|Vercel / Render Environment Variables|Baseline (Live Keys de Wompi, BD Producción).|

### 5. Gestión de Secretos

|Nivel de Entorno|Mecanismo|Acceso de Producción|Política de Rotación|
|---|---|---|---|
|Desarrollo|Local `.env`|N/A|Manual|
|Staging|PaaS Secrets (Vercel/Render)|N/A|Manual|
|Producción|PaaS Secrets (Vercel/Render)|`lead_engineer`|Manual / Trimestral o ante rotura de llaves bancarias|
