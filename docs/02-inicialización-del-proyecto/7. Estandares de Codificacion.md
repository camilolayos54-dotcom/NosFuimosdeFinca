### 5. Convenciones de Nomenclatura

|Elemento|Convención|Ejemplo Correcto|Ejemplo Incorrecto|Tipo|
|---|---|---|---|---|
|Java Classes|`PascalCase`|`ReservaEntity`|`reserva_entity`|framework-imposed|
|Java Methods|`camelCase`|`bloquearFechasConcurrentes`|`BloquearFechas`|framework-imposed|
|Java Constants|`SCREAMING_SNAKE_CASE`|`MAX_RETRY_COUNT`|`MaxRetry`|framework-imposed|
|React Components|`PascalCase`|`CheckoutButton`|`checkout-button`|framework-imposed|
|DB Tables|`snake_case`|`finca_reservas`|`FincaReservas`|decisión de equipo|
|Files (JS/TS)|`lowercase-hyphen` o `camelCase`|`api-client.ts`|`API_Client.ts`|decisión de equipo|

### 6. Límites de Módulos

|Módulo|Propósito|Puede Depender De|Dependencias Prohibidas|Ejemplo de Violación|
|---|---|---|---|---|
|Frontend React|Capa visual y de captura|Backend API (via HTTP)|Direct Database|Frontend ejecutando query SQL a PostgreSQL|
|Backend Controllers|Enrutamiento REST / Webhooks|Services|Direct Repositories|Controller inyectando repositorio JPA directo sin lógica de servicio|
|Backend Services|Lógica transaccional pesada (Locks)|Repositories, Config|Controllers|Un servicio retornando un HTTP Response en lugar de una Entidad|

### 7. Convenciones de Mensajes de Commit

|Tipo|Uso|Ejemplo|
|---|---|---|
|feat|Nueva característica|`feat(reservas): add soft-lock logic`|
|fix|Corrección de error|`fix(auth): handle expired token`|
|chore|Mantenimiento|`chore(deps): update nextjs version`|
|docs|Documentation|`docs(api): update webhook payload format`|
|test|Cambios en pruebas|`test(db): add concurrency jmeter scenario`|
|refactor|Refactor|`refactor(ui): extract checkout summary`|
|perf|Rendimiento|`perf(db): add index on dates`|
|ci|CI changes|`ci(github): add pre-commit hooks`|
|Breaking|Cambio con ruptura (Breaking change)|`feat(api)!: remove v1 payment webhook`|
