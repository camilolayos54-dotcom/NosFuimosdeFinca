### 2. Análisis de Riesgos

|Categoría de Riesgo|Descripción de Riesgo|Tipo de Prueba Principal|
|---|---|---|
|Integrity & Concurrency (NFR-01)|Múltiples peticiones concurrentes superando el Soft-Lock creando dobles reservas.|Integration / Stress Test (DB Layer)|
|Reliability (NFR-03)|Fallo silencioso o timeout del Webhook bancario corrompiendo la expiración del inventario.|End-to-End (Webhooks) / Unit Tests|
|Business Logic Correctness|Errores en la liquidación de tarifas y matemática de fechas.|Unit Tests (Backend)|

### 3. Pirámide de Pruebas

|Tipo de Prueba|Proporción Objetivo|Justificación|
|---|---|---|
|Unit|60%|Algoritmos críticos aislados de liquidación y matemática de fechas requieren alta cobertura estática.|
|Integration|30%|Pruebas de concurrencia y estrés contra la BD son el foco central para validar NFR-01 (Row-Locks).|
|End-to-End|10%|E2E visual (Playwright) diferido. Solo E2E manual/scripts para Webhooks de Staging (Wompi).|
|Contract|0%|API interna única, sin consumo externo B2B actualmente.|

### 4. Política de Cobertura

|Campo|Valor|
|---|---|
|Minimum Line Coverage|70% (Backend Transactional Layer)|
|Justification if below 80%|Cobertura concentrada 100% en servicios core de reservas. Controladores y DTOs triviales excluidos del cálculo estricto.|
|Coverage Report Required|Por pull request|
|CI Failure Condition|Coverage below threshold en capa de servicios|

### 5. Convenciones de Archivos de Prueba

|Element|Convention|
|---|---|
|File Location Strategy|Colocated en frontend (`__tests__`), directorio dedicado `src/test/java` en backend.|
|File Naming|`[Name].test.ts` (FE), `[Name]Test.java` (BE)|
|Suite Naming|Debe reflejar la clase o componente a probar (`ReservaService`)|
|Case Naming|`methodName_StateUnderTest_ExpectedBehavior`|
|Prohibited Names|`testUser`, `testFindById`, `shouldWork`, `test1`|

### 6. Herramientas de Pruebas

|Propósito|Herramienta|Versión|
|---|---|---|
|Unit Testing|JUnit 5 + Mockito (BE) / Vitest + RTL (FE)|Más reciente|
|Integration Testing|TestContainers + Apache JMeter / K6|Más reciente|
|Contract Testing|N/A|N/A|
|End-to-End Testing|Scripts manuales de Ngrok / Wompi|N/A|
|Coverage Reporting|JaCoCo (Java)|Más reciente|

### 7. Cumplimiento de CI

|Condición|Comportamiento CI|
|---|---|
|Failed tests|Bloquear merge|
|Coverage below threshold|Bloquear merge (para capa transaccional)|
|Coverage report failure|Bloquear merge|
