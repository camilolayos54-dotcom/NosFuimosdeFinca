### 2. Definición de Preparado

|#|Condición|Método de Verificación|
|---|---|---|
|1|Requirement is written, scoped, and unambiguous (User Story format)|Una segunda persona puede leer y confirmar sin interpretación|
|2|Acceptance criteria are defined and testable (Min 2 bullets)|Una tercera persona puede verificar sin interpretación|
|3|Apoyo Visual UI / Figma link attached para cambios de Frontend|Verificación visual|
|4|All technical dependencies are identified and available (ej. Llaves API Wompi)|Lista de dependencias verificada|
|5|Task fits within a single iteration|Alcance confirmado antes de iniciar|
|6|Task is estimated|Estimación presente en el tracker|
|7|No known blockers|Campo de bloqueadores vacío|

### 3. Definición de Hecho

|#|Condición|Método de Verificación|
|---|---|---|
|1|Code passes all linting rules from D7 (ESLint / Checkstyle)|Automatizado — CI|
|2|Code is formatted per D7 formatter (Prettier)|Automatizado — pre-commit / CI|
|3|No linter errors (TS Compila `strict: true` sin `any`)|Automatizado — CI|
|4|Unit tests written for all new business logic|Revisión de código|
|5|Coverage meets threshold from D8: `70%` (Backend)|Automatizado — CI|
|6|All existing tests pass|Automatizado — CI|
|7|No skipped tests without documented reason|Revisión de código|
|8|Resiliencia Transaccional: Rutas de falla blindadas con `try/catch` y HTTP 409|Revisión de código|
|9|Review completed per role (Peer review o 24h solo cooldown)|Equipo: revisión de pares / Solo: lista de autoverificación|
|10|Review comments resolved or deferred with task|Revisión de código|
|11|Integración vía "Squash and Merge" sin dependencias cíclicas|Automatizado — Merge|
|12|ADR written and merged when architecture changes|Revisión de código|
|13|Acceptance criteria pass in staging|Verificación manual|
|14|No known P1 or P2 bugs introduced|Revisión de código + testing|

### 4. Límite de Cumplimiento

|Puerta|Aplicada Por|Aplicada Cuando|
|---|---|---|
|Definition of Ready|Executor|Antes de iniciar el trabajo|
|Definition of Done|Reviewer or executor|Antes del merge o lanzamiento|
|Process-level gate|P1-D16|Marco de flujo de trabajo exterior|
