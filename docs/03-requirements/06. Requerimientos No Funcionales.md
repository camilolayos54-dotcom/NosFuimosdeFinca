### 3. Lista Global de NFRs
| ID | Categoría | Scope (Módulo) | Descripción | Método de Medición Estricto |
| --- | --- | --- | --- | --- |
| NFR-001 | Performance | `MOD-PROP` | La carga inicial del catálogo (fotos webp y textos) debe renderizarse rápido en móviles. | Pruebas de Lighthouse: LCP (Largest Contentful Paint) < 2.5s en percentil 90 usando red 3G simulada. |
| NFR-002 | Performance | `MOD-RSV` | El endpoint de creación de reserva debe responder velozmente para evitar doble clicks. | Monitoreo APM: Response time de API < 500ms en percentil 95. |
| NFR-003 | Availability | `MOD-PAY` | Tolerancia a fallos de Wompi. Si la pasarela de pagos cae, la plataforma no debe fallar internamente. | Prueba Chaos Engineering: Al simular HTTP 500 de Wompi, el sistema debe aplicar Retry con Exponential Backoff sin afectar la DB. |
| NFR-004 | Security | `MOD-AUTH` | Protección de contraseñas de finqueros y agencias. | Análisis estático de código: Verificación estricta del uso exclusivo de la librería Argon2id. |
| NFR-005 | Security | `MOD-AUTH` | Protección de sesiones contra secuestros de sesión (XSS/CSRF). | Test de Integración: Validar que el token JWT expire en máximo 2 horas y obligatoriamente viaje en HttpOnly/Secure cookie. |
| NFR-006 | Scalability | `MOD-RSV` | Soporte de Overbooking (concurrencia masiva de reservas intentando apartar la misma fecha). | Load Test K6 en Staging: Disparar 50 Transacciones por Segundo (TPS) a la misma Finca; asegurar 0 deadlocks SQL y solo 1 reserva exitosa. |
| NFR-007 | Observability | `MOD-NOT` | Trazabilidad de fallos en envíos de WhatsApp. | Script automatizado de Logs: El 100% de Webhooks fallidos deben escribir un log estructurado JSON con severity=ERROR y el payload original. |
| NFR-008 | Compliance | Global | Protección de datos personales (Ley 1581 de Colombia). | Auditoría de esquema DB: Los campos PII (emails, celulares) en PostgreSQL deben usar encriptación AES-256 en reposo. |
| NFR-009 | Maintainability | `MOD-RSV` | Estándar de calidad del motor transaccional de reservas. | Pipeline CI/CD: El build fallará automáticamente si el Code Coverage (Unit Tests) del módulo es inferior al 80%. |
