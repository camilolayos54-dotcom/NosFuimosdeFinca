# Entregable 11 (D11): Especificación de Requisitos del Sistema (SRS)

**Proyecto:** Nos Fuimos de Finca
**Fase:** 3 — Ingeniería de Requisitos
**Estado:** Aprobado

### 2. Descripción General del Sistema
El sistema centraliza el inventario de fincas rurales y orquesta pagos asíncronos para garantizar que ninguna propiedad reciba más de un abono para las mismas fechas, conectando a Turistas con Propietarios de forma segura.

### 3. Actores y Roles
- **Turista**: Usuario final (demanda), B2C anónimo. Rol: `turista-guest`
- **Propietario**: Arrendador rural (oferta), B2B autenticado. Rol: `propietario-admin`
- **Administrador**: Personal interno, moderador global. Rol: `sys-admin`

### 4. Glosario de Dominio
- **Soft-Lock**: Bloqueo temporal y condicional de fechas mientras el usuario completa la transacción.
- **Hard-Lock**: Bloqueo persistente e inmutable de fechas tras confirmación de pago por webhook.

### 5. Descripción General de Módulos
1. `AUTH`: Autenticación y Gestión de Usuarios
2. `SRCH`: Búsqueda y Navegación
3. `RSV`: Gestión de Reservas
4. `PAY`: Pagos y Facturación
5. `POWN`: Panel del Propietario
6. `PADM`: Panel del Administrador
7. `FCNT`: Gestión de Contenido de Fincas
8. `REV`: Calificaciones y Reseñas
9. `NOT`: Notificaciones

### 6. Resumen de Elicitación
Se usaron entrevistas estructuradas, workshops JAD y análisis de documentación (Wompi). Coverage tests pasados para los tres actores principales.
