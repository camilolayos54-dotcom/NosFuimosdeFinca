## 2. Llamadas API/SDK Salientes

- **Proveedor:** Wompi (Bancolombia)
- **Método:** Drop-in Widget en el Frontend (Next.js). No usaremos SDK de backend para iniciar el pago, delegaremos la captura de la tarjeta al SDK de JS de Wompi (`https://checkout.wompi.co/widget.js`).
- **Payload Inicial (Frontend a Wompi):**
  ```json
  {
    "currency": "COP",
    "amountInCents": 20000000,
    "reference": "UUID_DEL_SOFT_LOCK",
    "publicKey": "pub_test_XXXXX",
    "redirectUrl": "https://nosfuimosdefinca.com/checkout/success"
  }
  ```

## 3. Definiciones de Webhooks Entrantes

- **Route:** `POST /api/v1/payments/webhook`
- **Schema:** Definido en el D7 OpenAPI (WompiWebhookPayload).
- **Validation Logic (Firma Criptográfica):**
  Para evitar ataques de denegación de servicio o *spoofing* de pagos, el backend DEBE calcular el hash SHA-256 concatenando `transaction_id + status + amount_in_cents + timestamp + WOMPI_EVENTS_SECRET`. Si el Hash calculado no coincide con el `signature.checksum` del payload entrante, se aborta y retorna 400.
