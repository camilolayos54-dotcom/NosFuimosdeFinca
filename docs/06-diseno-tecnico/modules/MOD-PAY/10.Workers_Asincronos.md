## 2. Colas y Cargas Útiles (Payloads)

La validación y mutación del pago es síncrona en el momento que llega el webhook. SIN EMBARGO, el envío de la notificación (voucher) debe encolarse.

- **Broker:** Spring `@Async` + ApplicationEventPublisher (Memoria RAM por ahora, RabbitMQ a futuro).
- **Queue Name:** `PaymentConfirmedEvent`
- **Job Payload (Internal Java Record):**
  ```java
  public record PaymentConfirmedEvent(
      UUID bookingId,
      String turistaEmail,
      String fincaTitle,
      BigDecimal amount
  ) {}
  ```
- **Worker Logic (En el Módulo MOD-NOT):**
  1. Escucha `PaymentConfirmedEvent`.
  2. Genera el PDF del Voucher.
  3. Llama a la API de SendGrid.

## 3. Programación de Cron Jobs
No se requieren Cron Jobs para este módulo. Los Webhooks operan de manera Event-Driven (Push), no en Polling (Pull).
