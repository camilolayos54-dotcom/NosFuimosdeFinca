## 3. Definiciones de Gestión de Estado

- **Server State (React Query / SWR):** 
  - `usePaymentStatusPolling(bookingId)` -> Una vez que el usuario cierra el Widget de Wompi, el Frontend no puede confiar en el Widget. Empieza a hacer Polling (llamadas GET cada 3 segundos) al endpoint `/api/v1/bookings/{id}` para verificar si el status cambió de `SOFT_LOCKED` a `HARD_LOCKED` (lo que significa que el Webhook del backend ya hizo su trabajo).
