## 3. Definiciones de Gestión de Estado

- **Local State (`useState`):** Las fechas seleccionadas temporalmente en el `DateRangePicker` antes de presionar el botón de enviar.
- **Server State (React Query / SWR):** 
  - `useFincaAvailability(fincaId)` -> Fetch GET para traer los días deshabilitados.
  - `useCreateSoftLock()` -> Mutación POST para intentar bloquear las fechas.
- **Global State (Zustand):** No es necesario un estado global complejo para la creación de reservas. El `bookingId` retornado por el Soft-Lock se pasa como prop al modal de Checkout (`MOD-PAY`).
