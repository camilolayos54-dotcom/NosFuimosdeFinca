# Entregable 7 (D7): Contratos API (OpenAPI)

**Proyecto:** Nos Fuimos de Finca
**Fase:** 6 — Diseño Técnico
**Módulo:** MOD-RSV
**Estado:** Aprobado

## 2. Especificación OpenAPI YAML

```yaml
openapi: 3.0.3
info:
  title: Nos Fuimos de Finca - Booking API
  version: 1.0.0
paths:
  /api/v1/bookings:
    post:
      summary: Crear un Soft-Lock de Reserva
      tags: [Bookings]
      security:
        - BearerAuth: []
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CreateBookingDTO'
      responses:
        '201':
          description: Soft-Lock creado exitosamente
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/BookingResponseDTO'
        '409':
          description: Conflicto (Double Booking)
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
components:
  schemas:
    CreateBookingDTO:
      type: object
      required: [fincaId, checkIn, checkOut]
      properties:
        fincaId:
          type: string
          format: uuid
        checkIn:
          type: string
          format: date
        checkOut:
          type: string
          format: date
    BookingResponseDTO:
      type: object
      properties:
        bookingId:
          type: string
          format: uuid
        status:
          type: string
          example: "SOFT_LOCKED"
        lockedUntil:
          type: string
          format: date-time
        totalPrice:
          type: number
    ErrorResponse:
      type: object
      properties:
        status:
          type: integer
          example: 409
        error:
          type: string
          example: "Conflict"
        message:
          type: string
          example: "Fechas no disponibles"
```
