## 2. Interfaces de Repositorio y Consultas Complejas

```java
public interface BookingRepository extends JpaRepository<Booking, UUID> {

    // QUERY CRÍTICA 1: Evitar Double Booking usando SELECT FOR UPDATE (Pessimistic Write Lock)
    @Lock(LockModeType.PESSIMISTIC_WRITE)
    @Query("SELECT b FROM Booking b WHERE b.fincaId = :fincaId " +
           "AND b.status IN ('SOFT_LOCKED', 'HARD_LOCKED') " +
           "AND (b.checkIn < :checkOut AND b.checkOut > :checkIn)")
    List<Booking> findOverlappingBookingsWithLock(
        @Param("fincaId") UUID fincaId, 
        @Param("checkIn") Date checkIn, 
        @Param("checkOut") Date checkOut
    );

    // QUERY 2: Encontrar Soft Locks expirados para el Async Worker
    @Query("SELECT b FROM Booking b WHERE b.status = 'SOFT_LOCKED' AND b.lockedUntil < CURRENT_TIMESTAMP")
    List<Booking> findExpiredSoftLocks();
}
```
