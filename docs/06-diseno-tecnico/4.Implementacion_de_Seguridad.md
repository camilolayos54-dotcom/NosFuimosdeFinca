## 2. Especificación de Criptografía y Tokens

- **Hashing de Contraseñas:** Se utilizará **Bcrypt** con un Work Factor (Rounds) de `10`. Ninguna contraseña viajará en texto plano en la BD.
- **Transmisión de Token:** El Frontend (Next.js) almacenará el token en un `HttpOnly Cookie` si se usa Server Components, o lo enviará en el Header `Authorization: Bearer <token>` si se usan Client Components.
- **Estructura del JWT (JSON Web Token):**
  - **Algoritmo:** HS256 (HMAC con SHA-256).
  - **Expiración:** 24 Horas (No habrá Refresh Tokens en V1 por simplicidad).
  - **Payload (Claims):** No incluirá datos sensibles (PII).
    ```json
    {
      "sub": "b2f6b8a1-c7d9-... ", // UUID del usuario
      "role": "PROPIETARIO",
      "iat": 1717200000,
      "exp": 1717286400
    }
    ```

## 3. Pseudocódigo / Lógica de Middleware

En el backend, configuraremos **Spring Security** con una cadena de filtros (Filter Chain) sin estado (`SessionCreationPolicy.STATELESS`).

### Flujo del `JwtAuthenticationFilter` (Middleware Base)
Este filtro se ejecuta en CADA petición HTTP que no sea pública (como Login o Buscar Fincas).

```text
FUNCION JwtAuthenticationFilter(request):
  1. Extraer el header "Authorization".
  2. SI (header NO ESTÁ o NO EMPIEZA con "Bearer "):
        -> Seguir la cadena (El usuario es ANÓNIMO).
  3. SI (header EMPIEZA con "Bearer "):
        token = ExtraerToken(header)
        INTENTAR:
           claims = DecodificarJWT(token, JWT_SECRET_KEY)
           SI (claims.exp < AHORA):
               -> Lanza TokenExpiredException (HTTP 401)
           SINO:
               // Construir el objeto de autenticación en memoria
               contexto = CrearContexto(claims.sub, claims.role)
               SpringSecurityContext.setAuthentication(contexto)
               -> Seguir la cadena
        CAPTURAR_ERROR:
           -> Retornar HTTP 401 Unauthorized
```

### Flujo del Control de Acceso basado en Roles (RBAC)
Las rutas estarán protegidas declarativamente usando anotaciones de Spring Boot (`@PreAuthorize`).

```java
// Ejemplo conceptual (No literal) del uso del RBAC
@RestController
@RequestMapping("/api/fincas")
public class FincaController {

    // Ruta Pública (Turistas y Anónimos)
    @GetMapping
    public List<FincaDTO> buscarFincas() { ... }

    // Ruta Protegida (Solo dueños pueden crear fincas)
    @PostMapping
    @PreAuthorize("hasRole('PROPIETARIO')")
    public FincaDTO crearFinca(@RequestBody FincaDTO dto) { ... }
    
    // Ruta Súper Protegida (Solo Admins aprueban fincas para publicarse)
    @PatchMapping("/{id}/approve")
    @PreAuthorize("hasRole('ADMIN')")
    public void aprobarFinca(@PathVariable UUID id) { ... }
}
```

### Seguridad Específica: Webhooks de Wompi (MOD-PAY)
La ruta `/api/payments/webhook` debe ser pública (Wompi no envía JWT), pero debe validar criptográficamente la firma para asegurar que nadie finge un pago.

```text
FUNCION ValidarWompiWebhook(request_body, firma_wompi):
  1. CadenaCruda = request_body.transaccion_id + request_body.status + request_body.monto + WOMPI_EVENTS_SECRET
  2. FirmaEsperada = SHA256(CadenaCruda)
  3. SI (firma_wompi === FirmaEsperada):
        -> Procesar Pago
     SINO:
        -> Lanzar SecurityException (HTTP 400 Bad Request)
```
