# Entregable 4 (D4): Sistema de Diseño y UI Kit

**Proyecto:** Nos Fuimos de Finca
**Fase:** 4 — Modelado del Sistema
**Estado:** Aprobado

### 1. Filosofía de Diseño
El sistema prioriza la **Confianza** y la **Velocidad**. Al tratar con pagos para alquileres rurales (sujetos históricamente a fraude), la interfaz debe lucir institucional, clara y responsiva (Mobile-First 90%).

### 2. Tokens de Diseño (Variables Centrales)

- **Colores:**
  - Primario: `Green-600` (Naturaleza, Fincas, Confianza).
  - Secundario: `Amber-500` (Llamados a la acción, Reservas).
  - Fondo: `Slate-50` (Fondo general) / `White` (Tarjetas).
  - Texto: `Slate-900` (Primario), `Slate-500` (Secundario).
  - Estado: `Red-500` (Error/Ocupado), `Yellow-400` (Soft-Lock/Pendiente).
- **Tipografía:**
  - Encabezados: `Inter` o `Outfit` (Moderno, geométrico, alta legibilidad).
  - Cuerpo: `Roboto` o `Inter`.
- **Espaciado (Base 4px/8px):**
  - Padding global móvil: `16px`.
  - Ancho máximo contenedor de escritorio: `1280px`.

### 3. Mapeo Atómico de Componentes UI

| Átomo / Molécula | Contexto de Uso | Reglas de Comportamiento |
| --- | --- | --- |
| **Finca Card (Molecule)** | Resultados de búsqueda. | Debe incluir carrusel de imágenes, Precio/Noche grande, y badge de "Verificada". Toque en la tarjeta abre el Detalle. |
| **Date Picker (Organism)** | Detalle de Finca, Checkout. | Fechas en `Bloqueo Permanente` están deshabilitadas (gris/tachado). Fechas en `Bloqueo Temporal` están amarillas (no seleccionables, tooltip: "En proceso de pago"). |
| **Book Button (Atom)** | CTA principal en Detalle. | Pasa a estado `Cargando` y bloquea múltiples clics para evitar race-conditions en el cliente antes de la solicitud de reserva. |
| **Status Badge (Atom)** | Dashboard Propietario/Admin. | Verde = Aprobada, Gris = Pendiente, Rojo = Suspendida. |

### 4. Accesibilidad y Reglas Móviles (WCAG)
- **Objetivos de Toque:** Todo botón (especialmente en Checkout y Calendario) debe tener al menos `44x44px` de área clickeable.
- **Contraste:** Texto secundario (`Slate-500`) sobre fondo blanco debe superar ratio 4.5:1.
- **Retroalimentación:** Todo error de validación en la pasarela o de concurrencia ("Alguien reservó antes que tú") debe mostrar un Toast/Banner explícito y ofrecer nuevas fechas, no solo fallar en silencio.
