### 1. SSD: Creación de Soft-Lock (Turista)

Este diagrama modela cómo se reserva una finca temporalmente para pasar al pago, garantizando cero colisiones.

```mermaid
sequenceDiagram
    autonumber
    actor Turista
    participant Web (Next.js)
    participant API (Spring Boot)
    participant DB (PostgreSQL)

    Turista->>Web: Clic en "Reservar" (Fechas X)
    Web->>API: POST /api/v1/reservas/soft-lock {fincaId, fechas}
    API->>API: Validar JWT Token
    
    API->>DB: BEGIN TRAN
    API->>DB: SELECT * FROM reserva WHERE finca_id=? AND fechas superpuestas FOR UPDATE
    DB-->>API: (Retorna registros conflictivos)
    
    alt Hay conflictos (Otra reserva viva)
        API->>DB: ROLLBACK
        API-->>Web: 409 Conflict ("Fechas tomadas")
        Web-->>Turista: Mostrar error y refrescar calendario
    else Fechas libres
        API->>DB: INSERT INTO reserva (estado='soft_lock', expires_at=now+15m)
        API->>DB: COMMIT
        API-->>Web: 201 Created {reservaId, montoAnticipo}
        Web-->>Turista: Redirigir a Pantalla de Checkout
    end
```

### 2. SSD: Cron Job Purgador de Soft-Locks Expirados

```mermaid
sequenceDiagram
    autonumber
    participant Cron (Spring Scheduler)
    participant API (Spring Boot)
    participant DB (PostgreSQL)

    loop Cada 5 minutos
        Cron->>API: purgeExpiredReservations()
        API->>DB: UPDATE reserva SET estado='cancelada' WHERE estado='soft_lock' AND expires_at < now()
        DB-->>API: Affected rows: N
    end
```
