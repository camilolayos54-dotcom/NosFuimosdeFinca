# Entregable 5 (D5): Mockups de Alta Fidelidad (MOD-RSV)

**Proyecto:** Nos Fuimos de Finca
**Fase:** 4 — Modelado del Sistema
**Módulo:** MOD-RSV (Gestión de Reservas)
**Estado:** Aprobado

*(Nota: En este artefacto textual, definimos las reglas de renderizado de la UI de alta fidelidad que el frontend aplicará según el UI Kit del D4).*

### 1. Restricciones de Mockup: Checkout Screen
- **Fondo:** `Slate-50`.
- **Card Detalle:** Fondo `White`, borde redondeado `8px`, sombra `lg`.
- **Temporizador Soft-Lock:** 
  - El texto "15:00 min" debe tener fuente monoespaciada para evitar saltos.
  - El icono del reloj y texto deben ser color `Amber-600` sobre fondo `Amber-50` para denotar urgencia.
- **Botón "Ir a pagar":** 
  - Fondo `Green-600`, texto blanco en `bold`.
  - Icono de candado para denotar seguridad en el pago.

### 2. Interacciones y Micro-animaciones
- Cuando el usuario toca una fecha ocupada en el calendario, esta hará una animación de "Shake" (200ms) y arrojará un Haptic Feedback (si es web mobile PWA) o cambiará brevemente a color `Red-500` para indicar error.
- Al tocar "Reservar", el botón entrará en estado `Loading` (Spinner) para esperar a que el Backend verifique la disponibilidad de último milisegundo y ejecute el Soft-Lock en PostgreSQL antes de pasar al Checkout.
