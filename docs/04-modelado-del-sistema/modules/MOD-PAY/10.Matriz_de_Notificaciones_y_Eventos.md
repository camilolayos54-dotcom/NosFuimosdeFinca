# Entregable 10 (D10): Matriz de Notificaciones y Eventos (MOD-PAY)

**Proyecto:** Nos Fuimos de Finca
**Fase:** 4 — Modelado del Sistema
**Módulo:** MOD-PAY (Pagos y Facturación)
**Estado:** Aprobado

### 1. Mapeo de Eventos a Notificaciones

| Disparador de Evento de Sistema | Canal | Audiencia (Rol) | Asunto / Propósito | Contenido (Variables) |
| --- | --- | --- | --- | --- |
| `PAGO_DECLINADO` | UI Toast / Banner | `turista-guest` | Notificar rechazo y motivar re-intento. | Mensaje del banco adquirente (Ej. "Fondos insuficientes"). No se envía email para evitar pánico. |
| `PAGO_ERROR_SISTEMA`| PagerDuty / Email | `sys-admin` | Alerta de integración caída con pasarela. | Endpoint fallido, Transaction ID, Response Code de Wompi. |
| `PAGO_APROBADO` | Internal Event Queue | M-03 (Reservas) | Disparar mutación a Bloqueo Permanente en BD. | `reservaId`, `monto` |

### 2. Event Sourcing Notes
El módulo de pago actúa como un productor de eventos ciego una vez valida el webhook. No envía correos directamente al Turista (esa responsabilidad de dominio se le delega al módulo M-03 Reservas o M-10 Notificaciones para mantener desacoplamiento).
