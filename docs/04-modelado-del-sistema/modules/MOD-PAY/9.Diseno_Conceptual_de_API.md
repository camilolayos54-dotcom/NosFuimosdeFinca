**Estado:** Aprobado

### 1. Endpoints API

| Método | Endpoint | Descripción | Autorización Requerida | Payload de Petición | Respuesta Exitosa |
| --- | --- | --- | --- | --- | --- |
| `POST` | `/api/v1/pagos/webhook/wompi` | Recibe eventos de Wompi (Asíncrono). | `Público` (Valida EventSecret) | `Wompi Webhook Event Object` | `200 OK` |
| `GET` | `/api/v1/pagos/referencia/:ref` | Permite al frontend verificar estado del pago (polling). | `turista-guest` (Dueño) | Ninguno | `200 OK` `{ estado: "aprobado", reserva_id }` |

### 2. Cargas de Error (Excepciones Esperadas)

- `401 Unauthorized` (Webhook Spoofing): 
  ```json
  {
    "error": "INVALID_SIGNATURE",
    "message": "La firma de eventos proporcionada no coincide con nuestro secreto criptográfico."
  }
  ```
- `422 Unprocessable Entity` (Polling before creation):
  ```json
  {
    "error": "PAYMENT_NOT_FOUND",
    "message": "No existe registro de pago para la referencia proporcionada."
  }
  ```
