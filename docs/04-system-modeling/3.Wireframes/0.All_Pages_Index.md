# Inventario Maestro de Páginas (All Pages Index)

**Project:** Nos Fuimos de Finca
**Phase:** 4 — System Modeling (D3)

Este documento es el **Mapa Cartográfico del Proyecto Frontend**. Enlista todas las pantallas físicas (Rutas URL) que existirán en la aplicación y declara qué Módulos Funcionales (`MOD-X`) le dan vida a cada una. 

La IA (Director UX) usará este índice para generar los Blueprints individuales en la carpeta `pages/`. El Diseñador Humano usará este índice para saber cuántas mesas de trabajo debe crear en Figma.

---

## Mapa Topológico (Sitemap Visual)

Este diagrama muestra la estructura de navegación y cómo las rutas anidadas (ej. el Dashboard) comparten un mismo Layout envolvente.

```mermaid
flowchart TD
    %% Raíz de la Aplicación
    Root((Dominio Principal))
    
    %% Ecosistema B2C
    subgraph B2C [Rutas Públicas B2C - Marketplace]
        direction TB
        Home[Home Page<br>Ruta: /]
        Search[Buscador<br>Ruta: /search]
        Finca[Perfil Finca<br>Ruta: /finca/slug]
        
        subgraph Embudo [Embudo de Pago Isolado]
            Checkout[Checkout<br>Ruta: /checkout/id]
            Success[Recibo<br>Ruta: /checkout/success]
            Checkout --> Success
        end
        
        Home --> Search
        Home --> Finca
        Search --> Finca
        Finca --> Embudo
    end
    
    %% Ecosistema B2B
    subgraph B2B [Rutas Privadas B2B - Portal Finqueros]
        direction TB
        HostLanding[Ventas Aliados<br>Ruta: /host]
        Login[Acceso<br>Ruta: /login]
        Onboarding[Muro KYC<br>Ruta: /onboarding/*]
        
        subgraph DashboardLayout [Layout Padre: Dashboard Sidebar]
            direction LR
            DashHome[Finanzas<br>Ruta: /dashboard]

            DashCal[Calendario<br>Ruta: /dashboard/calendario]

        end
        
        HostLanding --> Login
        Login --> Onboarding
        Login --> DashboardLayout
        Onboarding --> DashboardLayout
    end
    
    Root --> B2C
    Root --> B2B
```
