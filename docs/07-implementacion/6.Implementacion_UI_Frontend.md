# Entregable 6 (D6): Guía de Implementación de UI y Estado Frontend

**Proyecto:** Nos Fuimos de Finca
**Fase:** 7 — Implementación
**Estado:** Aprobado

## Propósito
Guía maestra para codificar la interfaz de usuario en Next.js siguiendo Feature-Sliced Design.

## Pasos de Ejecución para el Desarrollador

Debes iterar por la carpeta `frontend-app/src/features/`:

1. **Atoms & Molecules:** Revisa `P6-D9 (Frontend Architecture)`. Crea primero los componentes más pequeños (ej. `StarRating`, `DatePicker`) en la carpeta `components/` de cada feature.
2. **State Management:** Implementa el estado especificado en `P6-D9` sección 3.
   - Si dice *React Query*, crea un hook en `hooks/` (ej. `useSearchFincas.ts`) que apunte al endpoint correspondiente documentado en `P6-D7`.
   - Si dice *Zustand*, crea el store en `store/` (ej. `useBookingStore.ts`).
3. **Organisms:** Une los Atoms/Molecules en las secciones principales (ej. `BookingForm`, `FincaGrid`).
4. **Pages (App Router):** En `frontend-app/src/app/`, importa los Organisms para renderizar las rutas finales (`/fincas/[id]`, `/dashboard`, etc.). Usa TailwindCSS para el estilizado.
