# Entregable 5 (D5): Guía de Implementación de API Backend

**Proyecto:** Nos Fuimos de Finca
**Fase:** 7 — Implementación
**Estado:** Aprobado

## Propósito
Guía maestra para codificar la capa lógica del backend (Entidades, Repositorios, Servicios, Controladores).

## Pasos de Ejecución para el Desarrollador

Debes iterar por cada carpeta de módulo en `backend-monolith/src/main/java/com/finca/backend_monolith/modules/`:

1. **Capa Domain:** Lee `P6-D5 (Class Diagrams)` del módulo respectivo. Traduce el diagrama Mermaid a clases Java anotadas con `@Entity`. Respeta los tipos de datos (ej. `UUID`, `BigDecimal`).
2. **Capa Infrastructure:** Lee `P6-D6 (Data Access)`. Crea las interfaces extendiendo `JpaRepository`. Copia y pega las queries JPQL exactas, prestando especial atención al `@Lock(LockModeType.PESSIMISTIC_WRITE)` en el módulo `booking`.
3. **Capa API (Controladores):** Lee `P6-D7 (API Contracts)`. Crea los RestControllers mapeando los métodos HTTP (`GET`, `POST`) y las URLs exactas definidas en Swagger.
4. **Capa Application:** Usa inyección de dependencias (`@Service`) para conectar el Controlador con el Repositorio, implementando las reglas de negocio descritas en la Fase 4.
