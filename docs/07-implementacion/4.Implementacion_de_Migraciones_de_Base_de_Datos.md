# Entregable 4 (D4): Guía de Implementación de Migraciones de Base de Datos

**Proyecto:** Nos Fuimos de Finca
**Fase:** 7 — Implementación
**Estado:** Aprobado

## Propósito
Esta guía dicta cómo inicializar la base de datos PostgreSQL usando migraciones versionadas con Flyway.

## Pasos de Ejecución para el Desarrollador
1. Ve al archivo `P6-D2 (Physical Data Modeling)`.
2. En la carpeta `backend-monolith/src/main/resources/db/migration/`, crea un archivo llamado `V1__init_schema.sql`.
3. Copia TODO el contenido DDL de `P6-D2` (incluyendo la creación de tablas `users`, `fincas`, `bookings`, `payments`, etc.) en ese archivo.
4. En el mismo archivo, en la parte inferior, copia la sección `## 3. Indexing & Optimization Scripts` para crear los índices (especialmente `idx_bookings_dates` crucial para el Soft-Lock).
5. Ejecuta `mvn flyway:migrate` para correr el script contra la base de datos PostgreSQL local (configurada vía `docker-compose`).
