# Entregable 3 (D3): Guía de Ejecución de Walking Skeleton

**Proyecto:** Nos Fuimos de Finca
**Fase:** 7 — Implementación
**Estado:** Aprobado

## Contexto de Ejecución
El objetivo de este script es asegurar que el servidor backend básico pueda levantar sin errores de compilación y responda a una petición HTTP simple (Healthcheck), garantizando que las dependencias están correctamente vinculadas.

## Walking Skeleton Implementado
Se ha escrito el archivo: `backend-monolith/src/main/java/com/finca/backend_monolith/core/api/HealthController.java`.

Este controlador expone la ruta `/api/health`.

### Pasos de Verificación
Para verificar que el skeleton funciona, el desarrollador (o la IA) debe:
1. Abrir una terminal en `backend-monolith`.
2. Ejecutar `.\mvnw spring-boot:run`
3. En otra terminal, ejecutar `curl http://localhost:8080/api/health`
4. Se espera un JSON `{"status": "UP", "message": "Nos Fuimos de Finca API is running", "version": "1.0.0"}`.
