## 1. Reglas de Operación Estándar (SOP)

Antes de escribir la primera línea de código, debes apegarte a estas reglas. Si sigues esto a rajatabla, no tendrás problemas de arquitectura técnica a largo plazo.

### 1.1. Convenciones de Nombres
- **Backend (Spring Boot):**
  - Clases/Entidades/Interfaces: `PascalCase` (Ej. `BookingController.java`, `UserRepository.java`).
  - Variables/Métodos: `camelCase` (Ej. `findByEmail()`).
  - Paquetes: `minúsculas` (Ej. `modules.auth.domain`).
- **Frontend (Next.js):**
  - Componentes de React: `PascalCase` para empatar con el backend (Ej. `LoginForm.tsx`, `DatePicker.tsx`).
  - Lógica/Hooks/Stores: `camelCase` (Ej. `useAuthStore.ts`, `authApi.ts`).
  - Rutas de App Router: Siempre en minúsculas y organizadas en carpetas (Ej. `app/(auth)/login/page.tsx`).

### 1.2. Flujo de Trabajo y Definición de Terminado (DoD)
Nunca pases a la siguiente tarea (ticket) sin antes validar lo siguiente:
- **En Backend:** Tras guardar, ejecuta `.\mvnw clean compile` en la consola. Si falla, arréglalo de inmediato.
- **En Frontend:** Tras guardar, ejecuta `npm run lint`. No permitas que se acumulen warnings de ESLint o errores de tipado de TypeScript.

## 2. El Roadmap Paso a Paso

Sigue este roadmap estrictamente en orden. Está diseñado para que nunca necesites un componente que no haya sido programado previamente.

### Sprint 1: Infraestructura y Base de Datos

#### Ticket: `DB-01` - Migraciones SQL (Flyway)
- **Qué hacer:** Crear el script inicial de la base de datos con las tablas y los índices críticos (como el del Soft-Lock).
- **Dónde hacerlo:** Crea el archivo `backend-monolith/src/main/resources/db/migration/V1__init_schema.sql`.
- **Cómo saber qué poner:** Abre el documento `docs/06-technical-design/2.Physical_Data_Modeling.md` y copia los bloques DDL exactamente como están.
- **Siguiente paso:** Levanta el Docker con `docker-compose up -d` y comprueba la conexión a PostgreSQL.
