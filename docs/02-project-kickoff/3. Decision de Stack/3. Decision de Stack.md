# 3. Decisión de Stack

### 1. Metadatos del Documento
**Proyecto:** Nos Fuimos de Finca
**Fase:** 2 — Project Kickoff
**Entregable:** 3 de 13
**Estado:** Aprobado

### 2. Stack Tecnológico Oficial
| Capa | Tecnología Elegida | NFR que Satisface | ADR de Rechazo |
| --- | --- | --- | --- |
| Frontend | React 18 + Vite | NFR-01 (Performance) | Vue.js rechazado: El equipo no tiene experiencia demostrable (D8). |
| Backend / API | Node.js 20 + Express | NFR-01, NFR-05 | Python/Django rechazado: El event loop de Node.js es superior para el worker WhatsApp asíncrono (NFR-05). |
| Base de Datos | PostgreSQL 16 | NFR-01, NFR-03 | MongoDB rechazado: El modelo relacional garantiza integridad transaccional en reservas concurrentes. |
| Infraestructura | Railway (MVP) → AWS (Fase 5) | NFR-02 | AWS directo rechazado para el MVP: El equipo carece de experiencia DevOps en AWS (D8). Migración planificada. |
| Mensajería | WhatsApp Business API (Meta WABA) | NFR-05 | Sin alternativa real: Es el canal de comunicación dominante en el mercado objetivo rural colombiano. |

### 3. Implicación de Compuerta de Fase
- **¿Bloquea el avance?:** No.
- **Condición:** Proceed. Cada tecnología está justificada contra NFRs medibles y la capacidad real del equipo. Los ADRs de rechazo están documentados para prevenir debates en Fase 3.
