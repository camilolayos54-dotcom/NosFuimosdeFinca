### 2. Topología de Repositorio
- **Tipo:** Monorepo de Código (Polyglot).
- **Justificación:** Alojar el Backend en Java y el Frontend en JavaScript en el mismo repositorio permite sincronizar los releases, mantener la documentación centralizada en `docs/` y evitar fricciones de versionado para el equipo (un solo desarrollador / CTO).

### 3. Árbol de Carpetas Oficial
```text
nosfuimosdefinca-monorepo/
├── docs/                    # Requerimientos, framework de fases (F1, F2, F3)
├── backend/                 # Código fuente Java (API REST)
│   ├── src/main/java        # Lógica de dominio, Controladores, Servicios
│   ├── src/main/resources   # application.yml, scripts SQL
│   └── pom.xml / build.gradle
├── frontend/                # Cliente B2C Web en JavaScript
│   ├── src/                 # Lógica JS, HTML, CSS
│   └── package.json         # Dependencias (Vite, etc.)
└── .github/workflows/       # Pipelines de CI/CD (Build Java, Lints JS)
```

### 4. Reglas de Aislamiento
| Regla de Aislamiento | Impacto de Romper la Regla |
| --- | --- |
| **Separación de Responsabilidades:** El Frontend JS JAMÁS debe comunicarse directamente a la Base de Datos. | Violación de Seguridad. Inyección SQL directa desde el navegador del cliente. |
| **Gestión de Secretos:** Prohibido comitear llaves de Wompi o Meta WABA al código fuente (Backend o Frontend). Todo va por Variables de Entorno (`.env`). | Secuestro de infraestructura y pérdida de dinero B2B. |

### 5. Implicación de Compuerta de Fase
- **Condición:** Proceed. La estructura de carpetas asegura que el código fuente de las dos tecnologías base (Java y JS) vivan en sus propios entornos de construcción, orquestados por el mismo repositorio.
