### 2. Matriz Maestra de NFRs
| ID | Categoría ISO 25010 | Restricción Original | Métrica Cuantificada (Pasa/No Pasa) | Capa Responsable |
| --- | --- | --- | --- | --- |
| NFR-01 | Performance Efficiency | Fincas y turistas rurales tienen mala red 3G. | LCP < 2.5s en 3G Fast simulado. Peso total del Bundle JS Front < 1.5MB. | Frontend (JS) |
| NFR-02 | Reliability (Concurrency) | Colisiones por peticiones simultáneas de reservas. | API Java debe soportar locks pesimistas/optimistas con P95 < 400ms bajo 50 requests concurrentes. | Backend (Java) |
| NFR-03 | Usability / Accessibility | Las fincas odian portales web o descargar apps. | 0 logins requeridos para la finca B2B. Toda alerta B2B debe entregarse en su WhatsApp en < 10s tras la solicitud B2C. | Backend (Java/WhatsApp) |
| NFR-04 | Security | Protección de datos sensibles. | Cifrado TLS 1.3 en el transporte API. Nada de datos PCI en la Base de Datos. | Backend / DevOps |

### 3. Implicación de Compuerta de Fase
- **Condición:** Proceed. El NFR-01 limitará severamente el uso de frameworks pesados de JS en el Frontend. El NFR-02 justifica sobradamente la elección de un Backend robusto (Java) capaz de manejar alta concurrencia de hilos transaccionales para frenar el overbooking.
