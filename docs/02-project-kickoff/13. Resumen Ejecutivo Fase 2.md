### 1. Metadatos del Documento
**Proyecto:** Nos Fuimos de Finca
**Fase:** 2 — Project Kickoff
**Entregable:** 13 de 13
**Estado:** Aprobado

### 2. Identidad y Alcance (De D1)
- **Codename:** `nff-platform`
- **Mandato Central:** Construir una **Aplicación Web Transaccional (Software Real)** para solucionar el caos de las reservas rurales asíncronas.
- **Scope Fijo:** Frontend JS para captura web, Backend robusto Java para orquestar la concurrencia asíncrona (inversión de oráculo), BD Relacional, y WhatsApp API para alertas B2B a las fincas. 

### 3. NFRs Críticos (De D2)
- **Performance (JS):** LCP < 2.5s y peso inicial < 1.5MB (para tolerancia a red rural 3G).
- **Concurrencia (Java/SQL):** El Backend y la BD deben rechazar colisiones usando locks transaccionales robustos.

### 4. Stack Oficial (De D3 y D4)
| Capa | Tecnología Elegida | Justificación |
| --- | --- | --- |
| Frontend B2C | Vanilla JS (Vite) | Peso ultra-ligero y alta velocidad de carga. |
| Backend API | Java (Spring Boot / Javalin) | Poderosa gestión de concurrencia y POJOs para lógica de negocios B2B compleja. |
| Base de Datos | PostgreSQL | Integridad referencial y locks transaccionales absolutos (ACID). |
| Repositorio | GitHub | Bóveda de código fuente, CI/CD Pipeline. |

### 5. Reglas de Calidad y Cierre (De D7, D8, D9)
- **Nomenclatura (D7):** `snake_case` para PostgreSQL. Lints estrictos (ESLint/Checkstyle) antes del commit.
- **Testing Core (D8):** Pruebas Unitarias obligatorias (JUnit) sobre los flujos de reservas de Java. Pruebas E2E manuales en Staging para el Frontend.
- **Definition of Done (D9):** CI Verde, Código revisado y requisitos BDD comprobados localmente y en BD Staging.

### 6. Riesgos Técnicos Activos a Vigilar (De D12)
- **Colisión Concurrente:** Es la mayor amenaza. Si Java falla al bloquear el row en PostgreSQL, se genera el overbooking. Debe ser la máxima prioridad arquitectónica al inicio del desarrollo.
- **SPOF Distribución B2B:** El uso de la API de Meta requiere tolerancia a fallos en el código de envío (Retries y Dead Letter Queues).
