# 4. Estructura del Repositorio

### 1. Metadatos del Documento
**Proyecto:** Nos Fuimos de Finca
**Fase:** 2 — Project Kickoff
**Entregable:** 4 de 13
**Estado:** Aprobado

### 2. Tipo de Repositorio
- **Modelo:** Monorepo con npm Workspaces.
- **Justificación:** Equipo de 2 personas. Comparte tipos TypeScript entre Frontend y Backend. Reduce overhead de sincronización de paquetes.

### 3. Árbol de Directorios
```
nff-platform/
├── apps/
│   ├── web/                    # React 18 + Vite (Frontend B2C)
│   │   ├── src/
│   │   │   ├── booking/        # Flujo de reserva
│   │   │   ├── dashboard-finca/ # Panel del administrador de finca
│   │   │   ├── auth/           # Login / Registro
│   │   │   └── components/     # Componentes reutilizables
│   │   └── package.json
│   └── api/                    # Node.js 20 + Express (Backend)
│       ├── src/
│       │   ├── reservas/       # Dominio: Reservas (controller/service/repo)
│       │   ├── fincas/         # Dominio: Fincas
│       │   ├── auth/           # Dominio: Autenticación
│       │   └── notifications/  # Worker WhatsApp Asíncrono
│       └── package.json
├── packages/
│   └── shared/                 # Tipos TypeScript compartidos (DTOs)
├── infra/                      # Docker Compose + configs Railway
├── docs/                       # Documentación técnica
├── scripts/                    # Migraciones, seeds, utilidades
├── .gitignore
├── .env.example                # Plantilla sin valores reales
└── README.md
```

### 4. Archivos Raíz Obligatorios
- `.gitignore` — Excluye `.env`, `node_modules/`, `dist/`, `*.log`.
- `README.md` — Instrucciones de clonación, setup de variables de entorno y comandos de desarrollo.
- `.env.example` — Plantilla con todas las variables requeridas pero sin valores reales.

### 5. Implicación de Compuerta de Fase
- **¿Bloquea el avance?:** No.
- **Condición:** Proceed. La estructura es trazable al stack del D3. Los secretos están protegidos.
