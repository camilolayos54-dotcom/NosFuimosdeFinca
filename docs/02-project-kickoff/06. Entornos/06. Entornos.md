### 2. Mapa de Entornos Oficiales
| Entorno | Misión Principal | Componentes Activos |
| --- | --- | --- |
| Local (Dev) | Escritura de código en máquinas locales. | IDEs, Base de Datos local (Dockerizada o en memoria H2), Servidor local Java, Dev Server de Vite (JS). |
| Staging | Validar flujos de integración y Pull Requests antes de fusionar. QA Manual. | Servidor en PaaS (Railway/Render) de pruebas. DB PostgreSQL temporal. |
| Producción | Tráfico real. 100% SLA garantizado para agencias y turistas. | Instancia aislada de BD y Servidor Java + Hosting estático para JS. |

### 3. Aislamiento de Datos Externos (Contención)
| Servicio de Tercero | Regla para Local / Staging | Regla para Producción |
| --- | --- | --- |
| Base de Datos (PostgreSQL) | BD Inyectada con datos "Fake" (Mock data) en local y Staging. | Base de Datos ofuscada, aislada y sin puertos expuestos a red pública (Solo VPC). |
| Pasarela de Pagos (Wompi) | Uso obligatorio de Llaves Públicas/Privadas TEST (`pub_test_...` / `prv_test_...`). | Uso exclusivo de Llaves PROD (`pub_prod_...` / `prv_prod_...`). |
| Mensajería (Meta WABA) | Número Sandbox de Meta o Mock de servicio en Local. | Número Verificado Oficial. Autorizado para enviar mensajes a cualquier cliente. |

### 4. Implicación de Compuerta de Fase
- **Condición:** Proceed. El aislamiento garantiza que los programadores no modifiquen datos de producción mientras compilan Java en local.
