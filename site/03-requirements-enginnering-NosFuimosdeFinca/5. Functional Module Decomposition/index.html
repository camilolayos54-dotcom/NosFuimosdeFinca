    Project: Nos Fuimos de Finca Phase: 3 — Requirements Engineering Status: Approved

    2. Module Definitions

    M-01: Autenticación y Gestión de Usuarios

    • Module name: Autenticación y Gestión de Usuarios
    • Responsibility: Administra el ciclo de vida de la identidad digital, controlando el acceso seguro y la recuperación de credenciales.
    • Objectives: BG-01
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | propietario-admin | login, reset_pwd | Propietario | | sys-admin | login, force_reset | Administrador |
    • Domain concepts owned: Credenciales, Sesión JWT.
    • Dependencies: Ninguna. (Infraestructura base).
    • Out of scope: Edición del contenido de las fincas. Roles y permisos de la pasarela de pago.

    M-02: Búsqueda y Navegación

    • Module name: Búsqueda y Navegación
    • Responsibility: Expone el inventario público permitiendo filtrar propiedades por atributos y disponibilidad de fechas sin retener bloqueos.
    • Objectives: MV-01
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | turista-guest | view | Turista | | propietario-admin | view | Propietario | | sys-admin | view | Administrador |
    • Domain concepts owned: Motor de búsqueda, Filtros.
    • Dependencies:
    • M-07 (Incoming: Para leer fotos y metadata)
    • M-03 (Incoming: Para leer fechas Hard-Locked)
    • Out of scope: Bloqueo de fechas (eso es M-03). Modificación del inventario.

    M-03: Gestión de Reservas

    • Module name: Gestión de Reservas
    • Responsibility: Ejecuta el bloqueo atómico concurrente garantizando que una fecha no sea sobrevendida.
    • Objectives: BG-02
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | turista-guest | create | Turista | | propietario-admin | view, block | Propietario | | sys-admin | view, cancel_force | Administrador |
    • Domain concepts owned: Soft-Lock, Hard-Lock, Disponibilidad.
    • Dependencies:
    • M-04 (Incoming: Para transformar Soft en Hard).
    • Out of scope: Procesar la tarjeta de crédito. Emitir facturas.

    M-04: Pagos y Facturación

    • Module name: Pagos y Facturación
    • Responsibility: Intercepta la comunicación con la pasarela externa y procesa el webhook de validación financiera.
    • Objectives: MV-03
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | turista-guest | create (via Wompi) | Turista | | propietario-admin | view (recibos) | Propietario |
    • Domain concepts owned: Anticipo Financiero, Webhook de Pago.
    • Dependencies:
    • M-03 (Outgoing: Depende del Soft-Lock emitido por M-03).
    • Out of scope: Retención directa de datos PCI (tarjetas). Procesar el pago dentro del sistema.

    M-05: Panel del Propietario

    • Module name: Panel del Propietario
    • Responsibility: Agrega las métricas financieras y el estado general de ocupación en un tablero de control privado.
    • Objectives: MV-05
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | propietario-admin | view | Propietario |
    • Domain concepts owned: Tablero, Métrica de Ocupación.
    • Dependencies:
    • M-03 (Incoming: Agrega data de reservas)
    • M-04 (Incoming: Agrega data de pagos)
    • M-08 (Incoming: Agrega data de reseñas)
    • Out of scope: Edición masiva de la plataforma. Modificación de reservas.

    M-06: Panel del Administrador

    • Module name: Panel del Administrador
    • Responsibility: Centraliza el poder de moderación de disputas y la suspensión de entidades maliciosas.
    • Objectives: BG-03
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | sys-admin | view, moderate | Administrador |
    • Domain concepts owned: Suspensión, Resolución de Disputa.
    • Dependencies:
    • Todos los demás módulos (Outgoing: tiene potestad de lectura/escritura global).
    • Out of scope: Creación de fincas a nombre de un tercero (debe hacerlo el propietario).

    M-07: Gestión de Contenido de Fincas

    • Module name: Gestión de Contenido de Fincas
    • Responsibility: Almacena y distribuye la metadata multimedia y descriptiva del inmueble rural.
    • Objectives: MV-06
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | propietario-admin | create, edit | Propietario | | sys-admin | edit_status | Administrador |
    • Domain concepts owned: Finca, Galería de Imágenes, Amenidades.
    • Dependencies:
    • M-01 (Outgoing: Autorización del dueño).
    • Out of scope: Fijación de reglas financieras de cancelación. Gestión de reservas.

    M-08: Calificaciones y Reseñas

    • Module name: Calificaciones y Reseñas
    • Responsibility: Recolecta la valoración cuantitativa y cualitativa post-estadía para nutrir el motor de reputación.
    • Objectives: MV-07
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | turista-guest | create | Turista | | propietario-admin | reply | Propietario | | sys-admin | delete | Administrador |
    • Domain concepts owned: Reseña, Calificación.
    • Dependencies:
    • M-03 (Outgoing: Requiere que el estado de la reserva sea "Check-out realizado").
    • Out of scope: Moderación unilateral por parte del propietario (solo Admin puede borrar).

    M-09: Notificaciones

    • Module name: Notificaciones
    • Responsibility: Dispara eventos omnicanal ante mutaciones críticas del estado del sistema.
    • Objectives: MV-08
    • Actors and roles applicable: | Role | Permissions on this module | Actors holding this role | | --- | --- | --- | | system | execute | Sistema |
    • Domain concepts owned: Plantilla de Email, Evento Transaccional.
    • Dependencies:
    • M-03 (Outgoing: Observa eventos de reserva)
    • M-04 (Outgoing: Observa eventos de pago)
    • Out of scope: Lógica de negocio transaccional. Generación de PDFs.

    3. Candidate Requirement Mapping Table

    (As the source D4 file only provides a sample of requirements, this table maps the sample requirements to their corresponding modules)

    Session Ref Candidate Requirement Assigned Module
    NFF-001 The system shall emit a Soft-Lock expiration alert when TTL is breached for Turista with turista-guest role. M-09 (Notificaciones)
    NFF-002 The system shall display aggregated income metrics when requested for Propietario with propietario-admin role. M-05 (Panel Propietario)
    NFF-003 The system shall maintain Soft-Lock indefinitely until the Webhook is received or TTL polling fails. M-03 (Gestión de Reservas)
    NFF-002 The system shall disable an account and cancel unfulfilled bookings when triggered for Administrador with sys-admin role. M-06 (Panel Admin)

    (Note: In a full elicitation record, each module would have > 3 requirements. For this abbreviated MVP sample, we apply Conscious thin acceptance for modules with < 3 sampled requirements).
